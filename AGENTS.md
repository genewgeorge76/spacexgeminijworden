# AGENTS.md

Overview of the project structure for developers and AI agents working on this codebase.

## Project Overview

Marketing site for **J. Worden & Sons Paving & General Contracting** (`jwordenasphaltpaving.com`). Single-page React app with AI-backed estimator, interactive map/3D viewers, lead capture, and location-based landing pages. Deployed as a static bundle on Netlify with a handful of Netlify Functions.

### Tech Stack

| Layer | Technology |
|-------|------------|
| Build | Vite 7 |
| Frontend | React 19 |
| Routing | TanStack Router v1 (file-based, client-side) |
| Styling | Tailwind CSS 4 |
| 3D / Maps | Three.js, Leaflet |
| PDF | @react-pdf/renderer |
| AI | @anthropic-ai/sdk (direct) |
| Server | Netlify Functions (Node) |
| Language | TypeScript 5 (strict mode) |
| Deployment | Netlify (static SPA) |

This is a **Vite SPA**, not TanStack Start. There is no SSR and no file-based server routes — server-side code lives in `netlify/functions/` and is called by the client via `/.netlify/functions/*`.

## Key Concepts

### Client-side routing

Routes live in `src/routes/` and are compiled into `src/routeTree.gen.ts` by `@tanstack/router-plugin` (driven by `vite.config.ts`). The tree is consumed in `src/router.tsx`, wrapped with scroll restoration, and mounted from `src/main.tsx` into `#root` in `index.html`.

Because this is an SPA, `netlify.toml` includes a `/*  →  /index.html  200` rewrite so deep-linked routes are served the bundle.

### Server endpoints

Anything that needs server credentials or cross-origin calls goes in `netlify/functions/`. Client code calls these via `/.netlify/functions/<name>`. `ContactForm.tsx` and `LeadCaptureForm.tsx` are the current callers.

Edge functions live under `netlify/edge-functions/`. `geo-intercept` handles `/strike/*` — it reads `context.geo` and annotates the SPA response with `x-worden-geo-*` headers for regional rapid-dispatch landing pages. The route is registered in the `[[edge_functions]]` block in `netlify.toml`.

### AI

`src/ai/JWordenAIEngine.ts` talks to the Anthropic SDK directly. For a Netlify-hosted deployment, secrets are injected via environment variables configured in the Netlify UI — see *Environment Variables* below.

## Configuration Files

| File | Purpose |
|------|---------|
| `vite.config.ts` | React + TanStack Router plugin + Tailwind; `manualChunks` splits vendor bundles (react, router, leaflet, pdf, icons) |
| `tsconfig.json` | ES2022, strict, `@/*` → `src/*`, `noUnusedLocals`, `noUnusedParameters` |
| `netlify.toml` | `vite build` → `dist`, Node 20, SPA fallback redirect |
| `.github/workflows/` | `ai_lead_gatekeeper.yml` (hourly), `quality-gate.yml`, `weekly-seo-report.yml` |

## Development Commands

```bash
npm install          # Install dependencies
npm run dev          # Vite dev server
npm run build        # Production build → dist/
npm run preview      # Preview the dist/ build
npm run build-sitemaps  # Regenerate sitemap.xml
```

Local full-stack emulation with functions:

```bash
netlify dev          # http://localhost:8888 with functions proxied
```

## Conventions

### Naming
- Components: PascalCase (`LeadCaptureForm.tsx`)
- Utilities / hooks: camelCase (`estimator-engine.ts`, `useVoiceInterface.ts`)
- Routes: kebab-case files mapping to kebab-case URLs

### Styling
- Tailwind CSS utility classes
- Design tokens via CSS variables in `src/index.css`

### TypeScript
- Strict mode, `noUnusedLocals`, `noUnusedParameters`
- Import paths use the `@/` alias
- Prefer `import type` for type-only imports

### State
- Local state via React hooks; no global store currently
- Context (`src/lib/auth-context.tsx`) for auth where needed

## Environment Variables

Set on Netlify (or `.env` for local dev — see `.env.example`):

- `ANTHROPIC_API_KEY` — Claude API access for `src/ai/*`
- `VITE_SITE_URL` — canonical site origin, baked into the client at build time (defined in `netlify.toml`)

Any other AI provider keys or CRM webhook secrets should live in Netlify environment variables and be read only from `netlify/functions/` code, never from `src/`.
